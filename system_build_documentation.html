1. System Overview & Architecture

The B2B Logistics & Tracking System is an enterprise-grade corporate platform designed for robust end-to-end management of B2B supply chains. Built upon modern web architectural principles, it manages onboarding workflows, master catalogs, transaction management, driver assignment tracking, real-time dispatch, proof-of-delivery (POD) tracking, and reverse logistics.

Leveraging the power of Django 6.0.3 and Django REST Framework (DRF) 3.17.1, the platform ensures rapid serialization, consistent data validations, soft-isolated multi-tenant security structures, and robust token-based JWT authentication controls.

3. Secure Multi-Tenancy & Tenant Isolation

Strict data segregation is enforced programmatically at the REST controller layer using the CompanyScopedMixin. This guarantees complete soft isolation, preventing cross-tenant leakage.

When an authenticated user requests resources, the request's token is evaluated to determine their bound company_id. The mixin overrides the default get_queryset() method, ensuring queries are scoped exclusively to their own organization: 

class CompanyScopedMixin:
    """
    Enforces strict multi-tenant data isolation on DRF ViewSets.
    Filters the active model queryset by the requesting user's Company.
    """
    def get_queryset(self):
        user = self.request.user
        if user.is_authenticated and hasattr(user, "company"):
            return super().get_queryset().filter(company=user.company)
        return super().get_queryset().none()

Granular authorization is achieved using custom DRF permissions:

  • IsAdmin: Strict admin-only system configuration rights.
  • IsAdminOrManager: Operations manager and supervisor rights.
  • IsAdminManagerOrClient: Allows clients to inspect their own catalogs, orders, and returns.
  • IsDriver: Tailored specifically for the mobile-dispatch endpoints, enabling drivers to start transits and upload POD signature links.

4. Business Logic State Machines

In order to preserve high operational integrity and consistent stock accounting, state transitions in B2B supply chains are managed via strict workflow engines:

1. Order Lifecycle Transition

1
Pending
Client places B2B order, stock reserved.
2
Confirmed
Admin reviews and approves order contract.
3
Packed
Warehouse picks, packs, and tags SKU stock.
4
Dispatched
Couriers route shipment; parent order syncs.
5
Delivered
Driver collects signature/POD. Final state.

2. Reverse Logistics Stock Recapture Flow

When a client initiates a return, it enters a structured verification pipeline. Stock recapture is conditionally executed upon final physical confirmation by warehouse managers:

  • "Good" Condition: Products are immediately re-ingested into the warehouse's stock_available pool, logging an IN ledger record in InventoryLog.
  • "Damaged" Condition: Products are routed to specialized quarantine pools. The platform logs a DAMAGE ledger record, ensuring stock levels remain completely reconciled with actual physically inspectable units.

5. Branded Django Administration Portal

A high-fidelity administrative panel was built using Django's core admin frameworks, registered exclusively under admin_site.

Visual Enhancements

  • Color-Coded Badges: Standard raw string states are rendered as beautiful, styled HTML spans (e.g. Pending is rendered as a clean amber pill; Delivered and Paid are highlighted in modern, low-saturation emerald).
  • Optimized Relationships: To avoid long-running dropdown queries, all large foreign key associations utilize raw_id_fields widgets.
  • Soft Delete Controls: Destructive operations are completely restricted. Custom admin actions (such as deactivate_users) soft-disable elements, keeping referential records safely intact.
  • Nested Admin Inlines: Orders list their items, and companies list their registered users natively on the parent details page.
